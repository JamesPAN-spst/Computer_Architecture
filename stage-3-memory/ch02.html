寄存器（Register）与计数器（Counter）

01N 位寄存器的构成

🎯 为什么研究这个问题

在上一章中，我们学习了 D 触发器（D Flip-Flop）如何在时钟边沿锁存 1 位数据。然而，现实中的数据——无论是一个 ASCII 字符（8 位）还是一条指令（32 位）——都远不止 1 位。我们需要一种结构，能够在同一个时钟边沿同时捕获多位数据，并在下一个时钟到来之前稳定地保持它们。

这就是"寄存器"的由来：将若干个 D 触发器并排放置，共享同一根时钟信号，就形成了最基本的 N 位并行寄存器。寄存器是 CPU 内部最快的存储单元，其读写速度直接决定了处理器的运算吞吐率。

此外，实际系统中并非每个时钟周期都要更新寄存器。例如流水线中的暂停（Stall）操作要求某些寄存器保持不变。因此，"使能（Enable）"控制和"并行加载（Parallel Load）"机制成为寄存器设计的核心话题。

🔍 抽象化方法

我们把 N 位寄存器看作一个黑盒，它有 N 条数据输入线 \(D_{N-1}, D_{N-2}, \ldots, D_0\)，N 条数据输出线 \(Q_{N-1}, Q_{N-2}, \ldots, Q_0\)，以及控制信号：时钟 CLK、使能 EN，以及可选的异步清零 CLR。

在 EN = 1 时，每个上升沿将 \(D_i\) 的值锁存到 \(Q_i\)；在 EN = 0 时，无论输入如何变化，输出保持不变。这相当于在每个触发器的 D 输入前加一个 2 选 1 多路选择器（MUX）：当 EN = 1 选择外部数据输入，当 EN = 0 选择自身输出反馈。

从系统级来看，寄存器是时序逻辑的"记忆单元"。组合逻辑负责"计算"，寄存器负责"保存中间结果"，两者交替工作构成了同步数字系统的基本范式。

📐 公式推导 / 模型建立

对于带使能控制的 N 位寄存器，每个触发器的次态方程为：

\[ Q_i^{+} = \text{EN} \cdot D_i + \overline{\text{EN}} \cdot Q_i \quad (i = 0, 1, \ldots, N-1) \]

当 EN = 1 时，\(Q_i^{+} = D_i\)，完成数据加载；当 EN = 0 时，\(Q_i^{+} = Q_i\)，状态保持。

寄存器的建立时间（Setup Time）\(t_{su}\) 和保持时间（Hold Time）\(t_h\) 要求数据在时钟边沿前后分别稳定一段时间。最高工作频率受限于：

\[ f_{max} = \frac{1}{t_{cq} + t_{comb} + t_{su}} \]

其中 \(t_{cq}\) 是触发器的时钟到输出延迟，\(t_{comb}\) 是组合逻辑的最长路径延迟。

⚠️ 建立时间违例（Setup Violation）是同步电路中最常见的时序错误。如果数据在时钟沿前 \(t_{su}\) 时间内仍在变化，触发器可能进入亚稳态（Metastable State），输出变得不确定。设计时必须进行静态时序分析（STA）确保所有路径满足约束。
🌐 实际应用

在 CPU 中，程序计数器（PC）、指令寄存器（IR）、通用寄存器文件（Register File）都是 N 位寄存器的实例。RISC-V 架构拥有 32 个 32 位通用寄存器（x0–x31），每个时钟周期可同时读取两个寄存器并写入一个寄存器。

在 I/O 接口中，数据寄存器用于暂存外设传入/传出的数据；状态寄存器标识设备是否就绪；控制寄存器用于配置工作模式。这些都是带使能控制的并行加载寄存器的变体。

FPGA 内部的可配置逻辑块（CLB）本质上就是查找表（LUT）+ 寄存器的组合，每个 CLB 中的触发器都可以配置为带使能、带同步/异步清零的寄存器。

✏️ 例题分析与易错点

题目：设计一个 4 位寄存器，要求在 EN = 1 且 CLK 上升沿时加载数据，EN = 0 时保持。若输入 D = 1010，初始状态 Q = 0000，画出两个时钟周期的时序波形。

❌ 常见错误：忘记在 EN = 0 的周期中保持输出，误认为输出随输入变化——这是把寄存器当成了透明锁存器（Latch）。

✅ 正确分析：第 1 个周期若 EN = 1，上升沿后 Q = 1010；第 2 个周期若 EN = 0，上升沿后 Q 仍为 1010，与 D 的当前值无关。

❌ 另一常见错误：将使能信号直接与时钟做与门（Clock Gating）而未加保护，导致时钟毛刺。实际设计中应使用专用的时钟门控单元（ICG Cell）来安全地门控时钟。

02移位寄存器（Shift Register）

🎯 为什么研究这个问题

并行传输需要 N 条数据线同时工作，但在很多场景下——如远距离通信、引脚数量有限的芯片封装——我们只能使用一两条数据线逐位传输。如何在串行（Serial）与并行（Parallel）之间进行转换，是移位寄存器要解决的核心问题。

移位寄存器（Shift Register）的基本思想非常简单：将多个触发器首尾相连，每个时钟周期将数据向同一方向"移动"一位。这一操作看似简单，却衍生出极其丰富的应用：串并转换、环形计数、伪随机序列生成（LFSR）、乘除 2 的快速运算等。

在现代 SoC 设计中，SPI、I²C、UART 等串行通信协议的核心硬件都离不开移位寄存器。理解移位寄存器的四种基本类型，是掌握数字通信接口的前提。

🔍 抽象化方法

根据输入方式（串行 S / 并行 P）和输出方式（串行 S / 并行 P），移位寄存器可分为四类：

SISO（Serial-In Serial-Out）：数据从一端串行输入，从另一端串行输出。本质上是一条"延迟线"，数据经过 N 个时钟周期后才到达输出端。

SIPO（Serial-In Parallel-Out）：串行输入，经过 N 个周期后所有位同时可用。这就是"串转并"的核心——SPI 从机接收数据时使用此结构。

PISO（Parallel-In Serial-Out）：先并行加载 N 位数据，然后逐位移出。这是"并转串"——UART 发送器的数据移出部分就是 PISO。

PIPO（Parallel-In Parallel-Out）：并行加载并并行输出，看似与普通寄存器相同，但加入移位功能后可实现快速乘/除 2 运算。

📐 公式推导 / 模型建立

以 4 位右移 SISO 为例，设串行输入为 \(S_{in}\)，各触发器状态为 \(Q_3, Q_2, Q_1, Q_0\)，则每个时钟沿的状态转移为：

\[ Q_3^{+} = S_{in}, \quad Q_2^{+} = Q_3, \quad Q_1^{+} = Q_2, \quad Q_0^{+} = Q_1 \]

串行输出 \(S_{out} = Q_0\)，即最低位移出。经过 4 个时钟周期，输入的 4 位数据完整地通过寄存器。

对于环形计数器（Ring Counter），将 \(Q_0\) 反馈到 \(S_{in}\)（即 \(S_{in} = Q_0\)），则 N 位环形计数器有 N 个有效状态，依次循环。其状态数为：

\[ \text{模值} = N \quad (\text{环形计数器}), \qquad \text{模值} = 2N \quad (\text{Johnson 计数器}) \]

Johnson 计数器（扭环计数器）的反馈为 \(S_{in} = \overline{Q_0}\)，状态数翻倍为 \(2N\)。

⚠️ 环形计数器的有效状态仅占全部可能状态的一小部分（N / 2^N）。若因噪声或上电进入非法状态，计数器可能永远无法回到合法序列。设计时必须加入自启动（Self-Starting）逻辑或初始化复位电路。
🌐 实际应用

SPI 通信：SPI 主机和从机各有一个移位寄存器，主机的 MOSI 线连接从机的串行输入，从机的 MISO 线连接主机的串行输入。每个时钟周期交换 1 位，8 个周期后完成一字节的全双工通信。

LED 驱动（如 74HC595）：这是一个带输出锁存的 8 位 SIPO 移位寄存器。仅需 3 根引脚（数据、时钟、锁存）就可以驱动 8 个 LED，且可级联扩展到任意数量。

LFSR（线性反馈移位寄存器）：通过特定位的异或反馈，可生成长度为 \(2^N - 1\) 的伪随机序列，广泛用于 CRC 校验、加扰（Scrambling）和内置自测试（BIST）。

✏️ 例题分析与易错点

题目：一个 4 位 SIPO 移位寄存器初始状态为 0000，串行输入序列为 1, 0, 1, 1（先输入 1）。画出 4 个时钟周期后各触发器的状态。

❌ 常见错误：搞混移位方向——不确定新数据是进入 \(Q_3\)（最高位）还是 \(Q_0\)（最低位）。这取决于具体设计，题目应明确说明。

✅ 正确分析（假设右移，新数据进入 Q3）：

初始：0000 → 时钟 1 后：1000 → 时钟 2 后：0100 → 时钟 3 后：1010 → 时钟 4 后：1101。最终 Q3Q2Q1Q0 = 1101。

❌ 另一易错点：将 SIPO 和 PISO 的作用搞混。记忆口诀："先说的是输入方式，后说的是输出方式"——SIPO 就是串行进、并行出。

03同步计数器与异步计数器

🎯 为什么研究这个问题

计数器（Counter）是数字系统中最基础的时序模块之一。它的本质功能是"记录事件发生的次数"——每来一个时钟脉冲，计数值加 1（或减 1）。从最简单的二进制递增到复杂的可编程定时器，计数器无处不在。

根据触发器的时钟驱动方式不同，计数器分为两大类：同步计数器（Synchronous Counter）中所有触发器共享同一时钟信号，状态同时翻转；异步计数器（Asynchronous / Ripple Counter）中后级触发器由前级输出驱动，状态逐级"涟漪"传播。

两种方式各有优劣：异步计数器结构简单但速度慢、有毛刺；同步计数器速度快、无毛刺但需要更多组合逻辑。在工程实践中，如何选择以及如何设计任意模值的计数器（Mod-N Counter），是数字设计工程师的必备技能。

🔍 抽象化方法

异步计数器：以最简单的 4 位二进制递增计数器为例，T 触发器 \(T_0\) 的时钟接系统 CLK，\(T_1\) 的时钟接 \(Q_0\)，\(T_2\) 的时钟接 \(Q_1\)，\(T_3\) 的时钟接 \(Q_2\)。所有 T 输入恒为 1。每次 \(Q_i\) 产生下降沿时，\(Q_{i+1}\) 翻转。

同步计数器：所有触发器共享同一 CLK。第 \(i\) 位翻转的条件是"所有低位均为 1"，即 \(T_i = Q_0 \cdot Q_1 \cdots Q_{i-1}\)。这需要额外的与门组合逻辑，但所有位同时翻转，没有累积延迟。

模-N 计数器：当需要计数范围不是 2 的幂（如模-10 的 BCD 计数器）时，需要加入反馈逻辑，在计数到 N 时强制复位到 0。实现方式包括同步预置法和异步清零法。

📐 公式推导 / 模型建立

对于 N 位同步二进制递增计数器，每一位的翻转条件：

\[ T_0 = 1, \quad T_i = \prod_{j=0}^{i-1} Q_j \quad (i \geq 1) \]

最大计数频率取决于最长路径。对于异步计数器，最坏情况传播延迟为：

\[ t_{ripple} = N \cdot t_{pd} \]

其中 \(t_{pd}\) 是单个触发器的传播延迟。因此异步计数器的最高频率为：

\[ f_{max}^{async} = \frac{1}{N \cdot t_{pd}} \]

对于同步计数器，关键路径是最宽的与门加一个触发器：

\[ f_{max}^{sync} = \frac{1}{t_{pd} + t_{AND}(N) + t_{su}} \]

当 N 很大时，可采用超前进位（Carry Lookahead）技术减少与门级数，类似于加法器的进位加速。

模-N 计数器需要 \(\lceil \log_2 N \rceil\) 个触发器，当 \(N \neq 2^k\) 时需检测状态 N 并复位：

\[ \text{触发器数} = \lceil \log_2 N \rceil \]
⚠️ 异步清零法设计模-N 计数器时，计数器会短暂经过状态 N 后再复位到 0，产生宽度约为一个门延迟的毛刺。如果后级电路对毛刺敏感，必须使用同步预置法——在状态 N−1 时预置下一状态为 0，避免出现非法状态。
🌐 实际应用

74HC393（异步计数器）：经典的双 4 位二进制纹波计数器，常用于低频分频和简单计数场合。由于纹波延迟，不适合高速应用。

74HC163（同步计数器）：4 位同步二进制计数器，支持同步加载、同步清零和使能控制。广泛用于地址生成器、定时器和状态机。

BCD 计数器（模-10）：在数字时钟、频率计等需要十进制显示的场合不可或缺。两个 BCD 计数器级联可计数到 99，三个可到 999，以此类推。

在现代 FPGA 设计中，计数器通常用 HDL 直接描述（如 Verilog 的 always @(posedge clk) count <= count + 1;），综合工具自动选择最优实现方式。

✏️ 例题分析与易错点

题目：设计一个模-6 同步递增计数器（0 → 1 → 2 → 3 → 4 → 5 → 0），使用 JK 触发器，画出状态转移表和电路图。

❌ 常见错误：忘记分析"无关状态"（Don't Care States，即 110 和 111）的行为。如果计数器因干扰进入这些状态，它是否能自动回到合法序列？不考虑这一点可能导致系统锁死。

✅ 正确方法：需要 3 个触发器（\(\lceil \log_2 6 \rceil = 3\)）。列出 0–5 的状态转移表，用卡诺图化简每个 JK 触发器的激励方程。同时检查状态 6（110）和 7（111）的次态是否能在有限步内回到合法序列。

❌ 另一易错点：混淆同步清零和异步清零。同步清零在下一个时钟沿生效，异步清零立即生效。使用异步清零做模-N 会产生毛刺窄脉冲。

一个 4 位异步纹波计数器，若单个触发器的传播延迟为 10 ns，则最坏情况下从 CLK 沿到最终输出稳定需要多长时间？

04计数器的应用——分频与时序控制

🎯 为什么研究这个问题

数字系统中往往只有一个主时钟源（Master Clock），但不同模块需要不同频率的时钟。例如，晶振提供 50 MHz 的主时钟，但 UART 模块需要 9600 Hz 的波特率时钟，LED 闪烁需要 1 Hz，PWM 生成需要精确的分频比。

计数器天然具有分频功能：一个模-N 计数器每 N 个输入脉冲产生 1 个输出脉冲，输出频率恰好是输入频率的 \(1/N\)。利用这一特性，我们可以从单一时钟源产生任意频率的时钟信号。

更进一步，计数器不仅可以分频，还可以生成复杂的时序控制信号——在特定计数值时产生使能、选通或中断信号，从而编排系统中各模块的工作顺序。这就是"时序控制器（Sequencer）"的基本原理。

🔍 抽象化方法

基本分频：模-N 计数器的最高位（或进位输出）每 N 个时钟翻转一次，输出频率为 \(f_{out} = f_{in} / N\)。若 N 为偶数，可直接取最高位获得 50% 占空比的方波；若 N 为奇数，则需要特殊处理才能得到对称方波。

预分频器（Prescaler）：在高频系统中，直接用一个大模值计数器分频会导致组合逻辑过于复杂。常见做法是级联多级小计数器，例如先用模-10 将 100 MHz 降到 10 MHz，再用模-10 降到 1 MHz，最后再按需分频。

时序控制：将计数器的输出与译码器（Decoder）结合，可在特定计数值时产生"独热码"控制信号。例如，4 位计数器 + 4-16 译码器可产生 16 个依次激活的时序信号，用于控制微指令的执行步骤。

数字时钟设计：将"秒脉冲（1 Hz）→ 模-60 秒计数 → 模-60 分计数 → 模-24 时计数"级联，配合 BCD 到七段译码器驱动数码管，就构成了一个完整的数字时钟。

📐 公式推导 / 模型建立

分频比公式：

\[ f_{out} = \frac{f_{in}}{N} \]

级联分频——若两个计数器模值分别为 \(N_1\) 和 \(N_2\)，总分频比为：

\[ f_{out} = \frac{f_{in}}{N_1 \times N_2} \]

以数字时钟为例，从 32.768 kHz 晶振到 1 Hz 秒脉冲：

\[ 32768 = 2^{15} \]

因此只需一个 15 位二进制计数器，取最高位输出即可得到 1 Hz 方波。这正是石英钟表芯片的工作原理。

对于奇数分频（如 ÷3），使用单个计数器只能得到非 50% 占空比输出。要获得 50% 占空比的 ÷3 输出，需要使用上升沿和下降沿各触发一路计数器，再将两路输出取或：

\[ f_{out} = \frac{f_{in}}{2N+1}, \quad \text{占空比} = 50\% \]
⚠️ 分频时钟不同于门控时钟（Gated Clock）。分频时钟由触发器输出产生，与主时钟有固定的相位关系，时序工具可以正确分析；而门控时钟由组合逻辑直接控制时钟通断，容易产生毛刺和时序违例。在 FPGA/ASIC 设计中，应尽量使用时钟使能（Clock Enable）代替门控时钟，并用专用的时钟管理模块（如 PLL/MMCM）进行分频。
🌐 实际应用

UART 波特率发生器：假设系统时钟为 50 MHz，目标波特率为 9600，则分频值为 \(50{,}000{,}000 / (16 \times 9600) \approx 326\)（16× 过采样）。一个模-326 计数器即可生成 UART 所需的采样时钟。

PWM（脉宽调制）生成：一个 N 位计数器从 0 计到 \(2^N - 1\) 循环，同时与比较值做比较：计数值小于比较值时输出高电平，否则输出低电平。改变比较值就改变了占空比，这就是数字 PWM 的基本原理。

看门狗定时器（Watchdog Timer）：一个大模值递减计数器从预设值开始递减，如果在倒数到 0 之前没有被"喂狗"（重载），就产生系统复位信号。这是嵌入式系统中防止软件跑飞的重要机制。

实时时钟（RTC）：从 32.768 kHz 晶振出发，通过 15 级二分频得到 1 Hz 秒脉冲，再级联秒、分、时计数器和日历逻辑，构成完整的实时时钟模块。

✏️ 例题分析与易错点

题目：用 10 MHz 主时钟和计数器设计一个能产生精确 1 秒定时中断的定时器。

✅ 分析：分频比 \(N = 10{,}000{,}000 / 1 = 10^7\)。需要 \(\lceil \log_2 10^7 \rceil = 24\) 位计数器。计数器从 0 计到 9,999,999 后产生中断并复位。

❌ 常见错误 1：计数到 10,000,000 而非 9,999,999 才复位——这会导致实际分频比为 10,000,001，定时不准确。正确做法：计数从 0 开始，到 \(N-1\) 时复位。

❌ 常见错误 2：使用异步计数器实现大模值分频。24 位异步计数器的累积延迟可达数百纳秒，在 10 MHz（周期 100 ns）下根本无法正常工作。必须使用同步计数器。

❌ 常见错误 3：忽略计数器溢出的边界条件。若使用 Verilog 编写 if (count == 10_000_000)，但计数器只有 24 位，最大值为 16,777,215，这个比较本身没问题，但如果位宽不够就会永远无法匹配。务必确认位宽足够表示目标值。

⚡ 快速检测
系统主时钟为 50 MHz，要产生 1 kHz 的方波输出，计数器的模值应为多少？
