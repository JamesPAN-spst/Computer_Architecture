从模拟到数字

01为什么选择二进制（Binary）？——噪声容限的精妙

🎯 为什么研究这个问题

前三章我们学习了电压、电流、电容、电感、二极管和晶体管。这些都是模拟世界（Analog World）的概念——信号可以取连续范围内的任何值。

但计算机是数字的（Digital）——它只理解 0 和 1。为什么不用十进制（0–9 十个电压级别）？为什么不用三进制（0、1、2）？答案不是"因为大家都这么做"，而是有深刻的物理原因。

这个选择的核心在于噪声容限（Noise Margin）——用更少的状态来换取更强的抗干扰能力。这是一个信息理论与电路物理的完美结合。

🔍 抽象化方法

假设我们的电路电压范围是 0–5V。如果用十进制：每个数字占 0.5V 的区间（0.0–0.5V 代表 0，0.5–1.0V 代表 1，……）。任何超过 0.25V 的噪声就可能导致误判——一个数字变成相邻的另一个。

如果用二进制：0–2.5V 代表"0"，2.5–5.0V 代表"1"。噪声需要超过约 1V 才会导致误判——抗干扰能力提高了 4 倍！

这就是二进制的核心优势：用信息密度换取可靠性。

定义噪声容限：信号在传输过程中可以叠加的最大噪声幅度，且不会导致接收端误判。

📐 公式推导 / 模型建立

定量分析：

设电压范围为 \(V_{range}\)，使用 \(M\) 个状态。每个状态的电压区间宽度为：

\[\Delta V = \frac{V_{range}}{M}\]

噪声容限约为 \(\Delta V / 2\)（噪声超过半个区间才会跨入相邻状态）：

\[NM \approx \frac{V_{range}}{2M}\]

信息量为 \(\log_2 M\) bit。每 bit 的噪声容限为：

\[\frac{NM}{\log_2 M} = \frac{V_{range}}{2M \log_2 M}\]

这个值在 \(M = 2\)（二进制）时最大！虽然 \(M = 2\) 每个符号只携带 1 bit，但每 bit 的可靠性最高。

香农（Shannon）的信息论告诉我们：在给定信噪比下，可以通过编码达到理论极限速率。实际工程中，二进制是实现简单且接近理论极限的最佳选择。

⚠️ 二进制并非唯一选择！有些特殊应用使用三进制甚至更高基数。例如，闪存（Flash Memory）中的 MLC（Multi-Level Cell）用 4 个电压级别存储 2 bit，TLC 用 8 个级别存储 3 bit。代价是读写速度更慢、寿命更短、错误率更高——正是因为噪声容限减小了。
🌐 实际应用

SSD 中的多级单元：SSD 用 NAND 闪存存储数据。SLC（1 bit/cell）最快最可靠，QLC（4 bit/cell, 16 电压级别）密度最高但最脆弱。这是噪声容限与存储密度的直接权衡。

PAM-4 通信：高速以太网（如 400GbE）使用 PAM-4（Pulse Amplitude Modulation, 4-level），4 个电压级别传输 2 bit/symbol。需要更复杂的纠错编码来弥补降低的噪声容限。

为什么 CPU 坚持二进制？CPU 需要极高的可靠性（错误率 < \(10^{-15}\)）和极高的速度（GHz 频率）。在这种条件下，二进制是唯一实用的选择。

✏️ 例题分析与易错点

例题：一个系统电压范围 0–3.3V，使用二进制。高电平定义为 2.0–3.3V，低电平定义为 0–1.0V。求高电平和低电平的噪声容限。

❌ 常见错误：忽略"禁区"（1.0–2.0V），直接用 3.3/2 作为噪声容限。

✅ 正确解法：

假设输出高电平最低 \(V_{OH} = 2.4\,\text{V}\)，输入高电平阈值 \(V_{IH} = 2.0\,\text{V}\)：

高电平噪声容限 \(NM_H = V_{OH} - V_{IH} = 2.4 - 2.0 = 0.4\,\text{V}\)

假设输出低电平最高 \(V_{OL} = 0.4\,\text{V}\)，输入低电平阈值 \(V_{IL} = 1.0\,\text{V}\)：

低电平噪声容限 \(NM_L = V_{IL} - V_{OL} = 1.0 - 0.4 = 0.6\,\text{V}\)

系统噪声容限取较小值：\(NM = \min(0.4, 0.6) = 0.4\,\text{V}\)。

⚡ 快速检测
SSD 中的 TLC 闪存用 8 个电压级别存储 3 bit/cell，相比 SLC（2 级别，1 bit/cell），其噪声容限约为 SLC 的？

02采样（Sampling）与量化（Quantization）

🎯 为什么研究这个问题

自然界的信号——声音、温度、光照——都是连续的模拟量。但计算机只能处理离散的数字量（有限个 0 和 1）。如何把连续信号变成离散数字？这需要两个步骤：

  1. 采样（Sampling）：在时间上离散化——按固定间隔取值
  2. 量化（Quantization）：在幅度上离散化——将连续值映射到最近的离散级别

音频 CD 用 44.1 kHz 采样率和 16-bit 量化；你的声音传到 Siri 用的是 16 kHz 采样和 16-bit 量化。这些数字是怎么确定的？为什么不能随意选择？

🔍 抽象化方法

采样是在时间轴上每隔 \(T_s = 1/f_s\) 秒"拍一张快照"。关键问题是：采样间隔多大才不会丢失信息？

奈奎斯特（Nyquist）和香农（Shannon）给出了答案：采样频率必须至少是信号最高频率的 2 倍。

\[f_s \geq 2f_{max}\]

这被称为奈奎斯特采样定理（Nyquist-Shannon Sampling Theorem）。如果违反这个条件，高频成分会"伪装"成低频信号（混叠，Aliasing），且无法恢复。

量化是在幅度轴上将连续值"四舍五入"到最近的离散级别。N-bit 量化有 \(2^N\) 个级别。

📐 公式推导 / 模型建立

采样定理的直觉推导：

一个频率为 \(f\) 的正弦波，每个周期至少需要 2 个采样点（一个在峰值，一个在谷值）才能确定其频率和幅度。如果采样点更少，你无法区分 \(f\) 和另一个较低频率的正弦波——这就是混叠。

量化分析：

设模拟信号范围为 \([0, V_{ref}]\)，N-bit 量化。

量化步长（LSB）：

\[\Delta = \frac{V_{ref}}{2^N}\]

量化误差范围：\(\pm \Delta/2\)。信噪比（SNR）：

\[\text{SNR} \approx 6.02N + 1.76\,\text{dB}\]

这意味着每增加 1 bit，SNR 提高约 6 dB（幅度精度提高一倍）。

例：16-bit 量化的 SNR ≈ 98 dB，这就是 CD 音频能达到如此高动态范围的原因。

⚠️ 采样定理要求 \(f_s > 2f_{max}\)，不是 \(f_s \geq 2f_{max}\)！严格来说，等于 \(2f_{max}\) 时，只有在完美条件下才能重建（需要无限长信号和理想低通滤波器），实际中总要留余量。音频 CD 的 44.1 kHz 对应 22.05 kHz 带宽，略高于人耳上限 20 kHz。此外，采样前必须使用抗混叠滤波器（Anti-aliasing Filter）去除超过 \(f_s/2\) 的频率成分。
🌐 实际应用

音频数字化：CD 音质 = 44.1 kHz × 16 bit × 2 声道 = 1411.2 kbps。电话语音 = 8 kHz × 8 bit = 64 kbps。采样率和量化位数直接决定了音质和数据量。

数字摄像：摄像机的图像传感器将每个像素的光强度量化为数字值。14-bit RAW 图像比 8-bit JPEG 保留了更多的亮度细节。

CPU 中的 ADC：微控制器（MCU）内置的 ADC 通常为 10–12 bit，用于读取传感器（温度、加速度、触摸）的模拟信号。

✏️ 例题分析与易错点

例题：一个信号包含频率成分从 DC 到 \(10\,\text{kHz}\)。若使用 \(15\,\text{kHz}\) 的采样率，会发生什么？

❌ 常见错误：认为"15 kHz > 10 kHz 所以足够了"。

✅ 正确分析：

奈奎斯特要求 \(f_s \geq 2 \times 10 = 20\,\text{kHz}\)。当前 \(f_s = 15\,\text{kHz}\)，不满足！

奈奎斯特频率 \(f_N = f_s/2 = 7.5\,\text{kHz}\)。高于 7.5 kHz 的信号成分（7.5–10 kHz）会混叠到 5–7.5 kHz 的范围内，产生失真。

解决方案：要么提高采样率到 ≥20 kHz，要么在采样前加低通滤波器截除 7.5 kHz 以上的成分。

⚡ 快速检测
人耳能听到的最高频率约为 20 kHz。CD 音频的采样率为 44.1 kHz，这满足奈奎斯特定理吗？

03ADC / DAC 原理

🎯 为什么研究这个问题

采样和量化是理论概念——实际实现它们的电路分别是 ADC（Analog-to-Digital Converter，模数转换器）和 DAC（Digital-to-Analog Converter，数模转换器）。

ADC 是数字世界的"入口"：麦克风信号、温度传感器输出、摄像头像素……所有模拟信号都必须经过 ADC 才能被 CPU 处理。DAC 是"出口"：CPU 处理完的数字音频必须经过 DAC 才能变成你耳朵能听的声波。

不同的 ADC 架构在速度、精度、面积和功耗之间做着不同的权衡。理解这些权衡，是理解混合信号（Mixed-Signal）芯片设计的基础。

🔍 抽象化方法

ADC 的核心任务：将模拟电压 \(V_{in}\) 转换为 N-bit 数字编码。

\(\text{数字输出} = \text{round}\left(\frac{V_{in}}{V_{ref}} \times 2^N\right)\)

主要 ADC 架构：

  • Flash ADC（并行比较型）：用 \(2^N - 1\) 个比较器同时比较，速度最快，面积最大
  • SAR ADC（逐次逼近型）：用二分搜索法逐 bit 确定，速度和精度中等，最常用
  • Sigma-Delta ADC（Σ-Δ型）：过采样+噪声整形，精度最高，速度较慢

DAC 的核心任务：将 N-bit 数字编码转换为模拟电压。

📐 公式推导 / 模型建立

SAR ADC 工作原理（二分搜索）：

1. 将 MSB（最高位）设为 1，其余为 0 → DAC 输出 \(V_{ref}/2\)

2. 比较器判断 \(V_{in}\) 与 \(V_{ref}/2\)：若 \(V_{in} > V_{ref}/2\)，MSB = 1；否则 MSB = 0

3. 设下一位为 1，再次比较……重复 N 次

N-bit SAR ADC 需要 N 个时钟周期完成一次转换。转换速率 \(f_{\text{sample}} = f_{\text{clk}}/N\)。

R-2R DAC 工作原理：

使用只含 R 和 2R 两种阻值的梯形电阻网络。每个 bit 控制一个开关：

\[V_{out} = V_{ref} \times \frac{D}{2^N}\]

其中 \(D\) 是数字输入的十进制值（0 到 \(2^N - 1\)）。例如 8-bit DAC，D = 128 时 \(V_{out} = V_{ref}/2\)。

Flash ADC 的面积问题：

N-bit Flash ADC 需要 \(2^N - 1\) 个比较器。8-bit 需要 255 个，10-bit 需要 1023 个。面积和功耗随 N 指数增长，所以 Flash ADC 通常限于 6–8 bit，用于超高速应用（如高速示波器）。

⚠️ ADC 的有效位数（ENOB）通常低于标称位数！一个标称 12-bit 的 ADC，由于噪声、非线性等因素，实际有效位数可能只有 10.5 bit。ENOB 通过 SINAD（信纳比）计算：\(\text{ENOB} = (\text{SINAD} - 1.76)/6.02\)。选择 ADC 时不能只看标称位数。
🌐 实际应用

手机音频：手机麦克风输出的模拟信号由内置 Sigma-Delta ADC 数字化（通常 24-bit），然后经 DSP 处理。播放音乐时，数字信号经 DAC 转换为模拟信号驱动扬声器。

微控制器：Arduino 的 ATmega328p 内置 10-bit SAR ADC，可以读取 0–5V 的模拟信号，分辨率约 4.9mV。

高速通信：5G 基站中的 ADC 需要在 GHz 采样率下工作，通常使用时间交错（Time-Interleaved）Flash ADC 阵列。

数字示波器：高端示波器使用 8-bit Flash ADC 以 20 GS/s（每秒 200 亿次采样）的速率捕获信号。

✏️ 例题分析与易错点

例题：一个 10-bit SAR ADC，参考电压 \(V_{ref} = 3.3\,\text{V}\)，时钟频率 \(f_{clk} = 10\,\text{MHz}\)。求：(a) 量化步长，(b) 最大采样率。

❌ 常见错误：忘记 SAR ADC 需要 N 个时钟周期完成一次转换，直接用时钟频率作为采样率。

✅ 正确解法：

(a) \(\Delta = V_{ref}/2^{10} = 3.3/1024 \approx 3.22\,\text{mV}\)

(b) \(f_s = f_{clk}/N = 10\,\text{MHz}/10 = 1\,\text{MS/s}\)（每秒 100 万次采样）

（实际中还需要额外时钟用于采样保持和设置，所以真实采样率更低一些。）

04数字信号的逻辑电平（Logic Level）与扇出（Fan-out）

🎯 为什么研究这个问题

我们已经决定使用二进制，也知道了如何用 ADC/DAC 在模拟和数字之间转换。但还有一个根本性的问题：数字电路中，什么电压算"1"？什么电压算"0"？

如果我们不严格定义这些规则，不同芯片之间就无法可靠通信。这就是逻辑电平标准（Logic Level Standard）存在的意义——它精确规定了输出和输入的电压范围、噪声容限和驱动能力。

此外，一个逻辑门的输出能驱动多少个输入？这就是扇出（Fan-out）——它限制了数字电路的连接方式和信号完整性。

🔍 抽象化方法

逻辑电平标准定义了四个关键电压参数：

  • \(V_{OH}\)：输出高电平最低电压（Output High minimum）
  • \(V_{OL}\)：输出低电平最高电压（Output Low maximum）
  • \(V_{IH}\)：输入识别为高电平的最低电压（Input High minimum）
  • \(V_{IL}\)：输入识别为低电平的最高电压（Input Low maximum）

关键约束：\(V_{OH} > V_{IH} > V_{IL} > V_{OL}\)。

这种设计确保输出的有效范围总是比输入的判定范围更窄——留出了噪声容限。

📐 公式推导 / 模型建立

噪声容限计算：

\[NM_H = V_{OH} - V_{IH}\] \[NM_L = V_{IL} - V_{OL}\]

系统噪声容限：\(NM = \min(NM_H, NM_L)\)

常见逻辑电平标准对比：

标准V_DDV_OHV_OLV_IHV_IL
TTL5V2.4V0.4V2.0V0.8V
CMOS 3.3V3.3V2.4V0.4V2.0V0.8V
LVCMOS 1.8V1.8V1.2V0.45V1.17V0.63V

扇出的推导：

假设驱动门输出低电平时的灌电流能力为 \(I_{OL}\)，每个负载门输入低电平时需要的电流为 \(I_{IL}\)：

\[\text{Fan-out} = \left\lfloor\frac{I_{OL}}{I_{IL}}\right\rfloor\]

对于 CMOS 逻辑，输入电流极小（仅充放电电容），扇出主要由 RC 延迟决定：

\[t_d \approx 0.7 \times R_{out} \times (C_{out} + n \times C_{in})\]

其中 \(n\) 是扇出数。扇出越大，延迟越大。

⚠️ 不同逻辑电平标准的芯片不能直接连接！例如 5V TTL 输出直接连 1.8V LVCMOS 输入，会超过后者的绝对最大额定电压，可能损坏芯片。不同电平之间需要使用电平转换器（Level Shifter）。即使电压兼容，也要检查噪声容限是否满足。
🌐 实际应用

芯片接口设计：现代 SoC 的 I/O pad 通常支持多种电平标准（通过可编程的供电电压和阈值），方便与不同芯片互连。FPGA 更是可以配置每个 I/O bank 的电平标准。

信号完整性：在高速数字设计（PCB 布线）中，走线上的阻抗不匹配会导致信号反射，使电压在 \(V_{IH}\) 和 \(V_{IL}\) 之间振荡——这就是为什么高速信号需要阻抗控制和端接电阻。

缓冲器（Buffer）：当扇出需求超过单个门的驱动能力时，插入缓冲器（本质上是两个反相器级联）来增强驱动能力。现代综合工具会自动插入缓冲器。

✏️ 例题分析与易错点

例题：一个 CMOS 逻辑门，\(V_{OH} = 3.0\,\text{V}\)，\(V_{OL} = 0.2\,\text{V}\)，\(V_{IH} = 2.0\,\text{V}\)，\(V_{IL} = 1.0\,\text{V}\)。求噪声容限。在传输线上叠加了 0.8V 的噪声，信号还可靠吗？

❌ 常见错误：混淆 \(NM_H\) 和 \(NM_L\) 的计算方向，或者忘记取较小值。

✅ 正确解法：

\(NM_H = V_{OH} - V_{IH} = 3.0 - 2.0 = 1.0\,\text{V}\)

\(NM_L = V_{IL} - V_{OL} = 1.0 - 0.2 = 0.8\,\text{V}\)

\(NM = \min(1.0, 0.8) = 0.8\,\text{V}\)

0.8V 噪声 = 恰好等于噪声容限的边界值。不可靠！工程上需要留至少 20% 的余量，即噪声应 < 0.64V。
