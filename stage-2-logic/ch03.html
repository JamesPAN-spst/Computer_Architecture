01半加器（Half Adder）与全加器（Full Adder）

🎯 为什么研究这个问题

布尔代数和逻辑门是"原材料"，但计算机的存在意义是计算。最基本的计算就是加法——事实上，减法、乘法、除法都可以归结为加法的变体。因此，用逻辑门实现二进制加法是从"纯逻辑"走向"真正计算"的第一步。

但二进制加法有一个关键难题：进位（Carry）。当两个 1 相加时，结果是 10（即 0 带进位 1）。如何用逻辑门处理进位，以及如何高效地处理多位加法中的进位传播，构成了数字电路设计中最经典的问题之一。

🔍 抽象化方法

将加法问题分为两个层次：

  • 半加器（Half Adder）：处理两个 1 位输入的加法，不考虑来自低位的进位。输出：求和位 \(S\) + 进位位 \(C\)。
  • 全加器（Full Adder）：处理两个 1 位输入 + 一个低位进位输入 \(C_{in}\) 的加法。输出：求和位 \(S\) + 进位输出 \(C_{out}\)。

从真值表出发：半加器有 2 个输入 → 4 行真值表；全加器有 3 个输入 → 8 行真值表。由真值表直接推导逻辑表达式，再用门电路实现。

📐 公式推导 / 模型建立

1. 半加器真值表与逻辑

ABSC
0000
0110
1010
1101

从真值表直接读出：

\[S = A \oplus B \quad(\text{异或})\]

\[C = A \cdot B \quad(\text{与})\]

半加器只需一个 XOR 门和一个 AND 门。

2. 全加器真值表与逻辑

全加器有三个输入 \(A, B, C_{in}\)：

\[S = A \oplus B \oplus C_{in}\]

\[C_{out} = A \cdot B + (A \oplus B) \cdot C_{in}\]

推导过程：\(C_{out}\) 可以理解为"要么 A 和 B 同时为 1 直接产生进位（\(A \cdot B\)），要么 A 和 B 恰好有一个为 1（\(A \oplus B\)），并且低位传来了进位（\(C_{in}\)）"。

3. 全加器 = 两个半加器 + 一个 OR 门

第一个半加器计算 \(A + B\)，得到部分和 \(S_1 = A \oplus B\) 和部分进位 \(C_1 = A \cdot B\)。第二个半加器将 \(S_1\) 与 \(C_{in}\) 相加，得到最终和 \(S = S_1 \oplus C_{in}\) 和进位 \(C_2 = S_1 \cdot C_{in}\)。最终进位 \(C_{out} = C_1 + C_2\)。

⚠️ 易错点：全加器的进位输出不能简单写成 \(C_{out} = A \cdot B + A \cdot C_{in} + B \cdot C_{in}\)——虽然这个表达式逻辑上等价，但它需要 3 个 AND 门和 1 个 3 输入 OR 门，比"两个半加器"的实现多用门。在实际芯片中，门数和连线复杂度直接影响面积和速度。
🌐 实际应用

多位加法器：将 \(n\) 个全加器串联，每个全加器的进位输出连到下一个全加器的进位输入，就构成了 \(n\) 位行波进位加法器（Ripple Carry Adder）。这是最直观的多位加法实现。

减法器：利用补码原理，\(A - B = A + \overline{B} + 1\)。只需将 B 取反并将最低位 \(C_{in}\) 置为 1，就用加法器实现了减法。

增量器：只需一个半加器链（因为 +1 操作中 B 始终为 0 或 1），用于程序计数器（PC）的 PC + 1 操作。

✏️ 例题分析与易错点

例题：用全加器计算 \(1011_2 + 0110_2\)

从最低位开始：

  • 位 0：\(1 + 0 + 0 = 1\)，\(S_0 = 1\)，\(C_1 = 0\)
  • 位 1：\(1 + 1 + 0 = 10\)，\(S_1 = 0\)，\(C_2 = 1\)
  • 位 2：\(0 + 1 + 1 = 10\)，\(S_2 = 0\)，\(C_3 = 1\)
  • 位 3：\(1 + 0 + 1 = 10\)，\(S_3 = 0\)，\(C_4 = 1\)

结果：\(10001_2 = 17_{10}\)（验证：\(11 + 6 = 17\) ✓）

常见错误

  • ❌ 忘记将上一位的进位传入当前位。二进制加法与十进制一样有进位传播！
  • ❌ 混淆半加器与全加器的使用场景：最低位可用半加器（无低位进位），但其余位必须用全加器。
  • ✅ 正确做法：始终从最低位（LSB）开始，逐位向高位计算，每位都带入上一位的进位。
⚡ 快速检测

02行波进位（Ripple Carry）与超前进位（Carry Lookahead）

🎯 为什么研究这个问题

全加器解决了 1 位加法问题，但现实中我们需要 32 位甚至 64 位加法器。最直观的方案是将 \(n\) 个全加器串联——这就是行波进位加法器（Ripple Carry Adder, RCA）。

问题在于：行波进位的延迟与位数成正比，\(O(n)\)。对于 64 位加法器，进位必须从第 0 位一路"涟漪"到第 63 位。在 GHz 级别的 CPU 中，这个延迟完全不可接受。

超前进位加法器（Carry Lookahead Adder, CLA）通过预测进位将延迟降至 \(O(\log n)\)，这是数字电路中"空间换时间"思想的经典体现。

🔍 抽象化方法

核心观察：每一位的进位 \(C_{i+1}\) 只依赖于两件事——

  • 生成（Generate）：第 \(i\) 位自身直接产生进位。当 \(A_i = B_i = 1\) 时，无论低位有没有进位，该位一定产生进位。定义 \(G_i = A_i \cdot B_i\)。
  • 传播（Propagate）：第 \(i\) 位能将低位进位"透传"上去。当 \(A_i\) 和 \(B_i\) 恰好有一个为 1 时，低位来了进位就会继续传上去。定义 \(P_i = A_i \oplus B_i\)。

关键递推：\(C_{i+1} = G_i + P_i \cdot C_i\)。将这个递推展开，就可以用纯组合逻辑（不需要等待进位传播）直接计算每一位的进位。

📐 公式推导 / 模型建立

1. 行波进位加法器（RCA）

结构：\(n\) 个全加器串联，\(C_{out,i}\) 连到 \(C_{in,i+1}\)。

延迟分析：最坏情况下，进位从位 0 传播到位 \(n-1\)：

\[T_{RCA} = n \cdot t_{carry} \quad (\text{其中 } t_{carry} \text{ 是单个 FA 的进位延迟})\]

2. CLA 的递推展开

从递推公式 \(C_{i+1} = G_i + P_i \cdot C_i\) 出发：

\[C_1 = G_0 + P_0 C_0\]

\[C_2 = G_1 + P_1 C_1 = G_1 + P_1 G_0 + P_1 P_0 C_0\]

\[C_3 = G_2 + P_2 G_1 + P_2 P_1 G_0 + P_2 P_1 P_0 C_0\]

\[C_4 = G_3 + P_3 G_2 + P_3 P_2 G_1 + P_3 P_2 P_1 G_0 + P_3 P_2 P_1 P_0 C_0\]

每个 \(C_i\) 都直接用 \(G_j, P_j, C_0\) 表示——只需 2 级逻辑门（AND-OR），延迟为常数！

3. 延迟分析

CLA 4 位块的延迟：\(T_{CLA-4} = t_{pg} + t_{AND-OR}\)（计算 G/P 一级 + 展开一级）。对于 16 位加法器，用 4 个 4 位 CLA 块组成两级 CLA，延迟为 \(O(\log_4 n)\)。

⚠️ 易错点：CLA 并非"零延迟"——展开公式中每一级的 AND/OR 门的扇入（fan-in）随位数增大。实际中 CLA 通常以 4 位为一块，多块再用 CLA 的思想进行层级连接（两级 CLA），而不是直接展开全部 64 位。
🌐 实际应用

现代 CPU 中的 ALU：实际处理器中的加法器通常采用混合设计——4 位 CLA 块内部用超前进位，块与块之间根据需求使用行波或再一级 CLA。

前缀加法器：更现代的方法如 Kogge-Stone、Brent-Kung 加法器，将 CLA 思想与并行前缀计算结合，在 \(O(\log n)\) 级延迟和可接受的面积之间取得更好平衡。

加法器选择器（Carry Select Adder）：预先用两套加法器分别假设 \(C_{in} = 0\) 和 \(C_{in} = 1\) 计算，当实际进位到来时用 MUX 选择正确结果，又是一种"空间换时间"策略。

✏️ 例题分析与易错点

例题：计算 4 位 CLA 的 \(C_4\)

设 \(A = 1011, B = 0110, C_0 = 0\)。

计算 G 和 P（从 LSB 开始）：

  • \(G_0 = 1 \cdot 0 = 0, P_0 = 1 \oplus 0 = 1\)
  • \(G_1 = 1 \cdot 1 = 1, P_1 = 1 \oplus 1 = 0\)
  • \(G_2 = 0 \cdot 1 = 0, P_2 = 0 \oplus 1 = 1\)
  • \(G_3 = 1 \cdot 0 = 0, P_3 = 1 \oplus 0 = 1\)

\(C_4 = G_3 + P_3 G_2 + P_3 P_2 G_1 + P_3 P_2 P_1 G_0 + P_3 P_2 P_1 P_0 C_0\)

\(= 0 + 1 \cdot 0 + 1 \cdot 1 \cdot 1 + 1 \cdot 1 \cdot 0 \cdot 0 + 1 \cdot 1 \cdot 0 \cdot 1 \cdot 0 = 1\)

所以 \(C_4 = 1\)，与行波计算结果一致。

常见错误

  • ❌ 混淆 \(P_i = A_i + B_i\) 和 \(P_i = A_i \oplus B_i\)。对于 CLA 进位计算两者等价，但对于求和输出必须用 XOR。
  • ❌ 忘记 CLA 展开式中各项是"或"关系（任一项为 1 即产生进位）。
⚡ 快速检测

03多路复用器（Multiplexer）与解码器（Decoder）

🎯 为什么研究这个问题

加法器解决了"计算"问题，但一个完整的数字系统还需要"选择"和"分发"数据。想象一个十字路口：多路复用器（MUX）像一个旋转的道闸，根据控制信号选择哪条路的车流通过；解码器（Decoder）则像一个邮局分拣员，根据地址将一封信送到正确的邮箱。

MUX 和 Decoder 是组合逻辑中最基本的"数据路由"组件，它们在 CPU 中无处不在：寄存器文件的读端口是 MUX，内存地址译码是 Decoder，指令译码器的核心也是 Decoder。

🔍 抽象化方法

多路复用器（MUX）：\(2^n\) 个数据输入、\(n\) 条选择线、1 个输出。功能：\(Y = I_{S}\)，其中 \(S\) 是选择线的值。

解码器（Decoder）：\(n\) 个输入、\(2^n\) 个输出。功能：恰好第 \(S\) 个输出为 1，其余为 0（独热码 / one-hot）。

两者的关系：解码器可以看作 MUX 的"逆操作"。解码器 + OR 门可以实现任意布尔函数（因为它直接对应最小项之和）。

📐 公式推导 / 模型建立

1. 2:1 MUX

\[Y = \bar{S} \cdot I_0 + S \cdot I_1\]

这就是 Shannon 展开定理的电路实现。

2. 4:1 MUX

\[Y = \overline{S_1}\,\overline{S_0} \cdot I_0 + \overline{S_1}\,S_0 \cdot I_1 + S_1\,\overline{S_0} \cdot I_2 + S_1\,S_0 \cdot I_3\]

3. n-2ⁿ 解码器

3 输入 8 输出解码器的输出：

\[Y_i = \prod_{j=0}^{n-1} S_j^{(b_j)} \quad \text{其中 } b_j \text{ 是 } i \text{ 的第 } j \text{ 位}\]

即 \(Y_i\) 对应输入 \(S\) 的第 \(i\) 个最小项。

4. 用 MUX 实现任意函数

对于 \(n\) 变量函数 \(F\)，用 \(2^n : 1\) MUX：将所有变量接选择线，数据输入接 \(F\) 的真值表值（0 或 1）。更高效的方法：用 \(2^{n-1} : 1\) MUX，最后一个变量通过 Shannon 展开出现在数据端。

⚠️ 易错点：解码器的使能端（Enable）。带使能端的解码器，当 EN = 0 时所有输出为 0。忽略使能端会导致"总有一个输出为高"的错误假设，在内存地址译码中可能导致多个芯片同时被选中。
🌐 实际应用

CPU 寄存器文件：读操作就是一个 MUX——用寄存器编号作为选择线，从 32 个寄存器中选出目标寄存器的值。

内存地址译码：高位地址通过解码器选择具体的内存芯片（片选信号 CS）。

指令译码：操作码（opcode）通过解码器生成控制信号，驱动 ALU、寄存器文件、数据通路中的各个 MUX。

七段显示：BCD 到七段解码器就是一个特殊的解码器，将 4 位 BCD 码转换为 7 段 LED 的驱动信号。

✏️ 例题分析与易错点

例题：用 8:1 MUX 实现 \(F(A,B,C) = \sum m(1,2,4,7)\)

将 A, B, C 接到选择线 \(S_2, S_1, S_0\)。将数据输入按最小项设置：

  • \(I_0 = 0\)（m0 不在集合中）
  • \(I_1 = 1\)（m1 在集合中）
  • \(I_2 = 1, I_3 = 0, I_4 = 1, I_5 = 0, I_6 = 0, I_7 = 1\)

常见错误

  • ❌ 选择线顺序搞反（\(S_0\) 接了最高位变量）。约定：\(S_0\) 对应最低位变量。
  • ❌ 解码器输出编号与最小项编号不对应。

04比较器与奇偶校验

🎯 为什么研究这个问题

加法器让我们能"计算"，MUX/Decoder 让我们能"路由"。但计算机还需要"判断"——两个数是否相等？谁大谁小？数据在传输中是否出错？

比较器回答前两个问题：它是分支指令（if/else）的硬件基础。奇偶校验回答第三个问题：它是最简单的错误检测机制，为后续学习 ECC（纠错码）等更高级方法奠定基础。

🔍 抽象化方法

1 位比较器：输入两个 1 位数 \(A, B\)，输出三个信号：\(A > B\)，\(A = B\)，\(A < B\)。

多位比较器：从最高位开始逐位比较，高位结果优先（类似字典序）。可以级联 1 位比较器实现。

奇偶校验：对所有数据位做 XOR，结果就是奇偶校验位。XOR 的本质是"计算 1 的个数的奇偶性"。

📐 公式推导 / 模型建立

1. 1 位相等比较

\[EQ = \overline{A \oplus B} = A \odot B \quad (\text{同或 / XNOR})\]

当 \(A = B\) 时 XOR 为 0，取反得 1。

2. N 位相等比较

\[EQ = \prod_{i=0}^{n-1}(A_i \odot B_i) = (A_0 \odot B_0) \cdot (A_1 \odot B_1) \cdots (A_{n-1} \odot B_{n-1})\]

所有位都相等时，总输出才为 1。

3. 大小比较（从高位到低位）

从最高位 \(n-1\) 开始：若 \(A_{n-1} > B_{n-1}\) 则 \(A > B\)；若 \(A_{n-1} < B_{n-1}\) 则 \(A < B\)；若相等，比较下一位。

级联公式：\(GT_{out} = (A_i > B_i) + (A_i = B_i) \cdot GT_{in}\)

4. 奇偶校验

\[P = D_0 \oplus D_1 \oplus D_2 \oplus \cdots \oplus D_{n-1}\]

偶校验：使总 1 个数为偶数（\(P_{even} = D_0 \oplus D_1 \oplus \cdots\)）。

奇校验：使总 1 个数为奇数（\(P_{odd} = \overline{P_{even}}\)）。

⚠️ 易错点：奇偶校验只能检测奇数个位翻转错误。如果同时翻转 2 位，校验位不变，错误无法被检测到。这就是为什么实际系统中使用 CRC、Hamming 码等更强的校验方法。
🌐 实际应用

CPU 分支判断：比较器的输出直接驱动 ALU 的零标志（Zero Flag）和负标志（Negative Flag），用于条件分支指令（BEQ, BNE, BLT 等）。

地址比较：缓存（Cache）中的标签比较器（Tag Comparator）用来判断请求地址是否命中。

数据完整性：DRAM 使用 ECC（纠错码），其核心思想建立在奇偶校验之上。PCIe、以太网帧使用 CRC 校验。

74 系列芯片：74HC85（4 位比较器）、74HC280（9 位奇偶校验器）是经典 TTL/CMOS 芯片。

✏️ 例题分析与易错点

例题 1：比较 \(A = 1010\) 和 \(B = 1001\)

从最高位开始：位 3 都是 1（相等）→ 位 2 都是 0（相等）→ 位 1：A₁=1, B₁=0 → A > B。

结果：\(1010_2 = 10 > 9 = 1001_2\) ✓

例题 2：求 \(10110\) 的偶校验位

\(P = 1 \oplus 0 \oplus 1 \oplus 1 \oplus 0 = 1\)。数据中有 3 个 1（奇数），所以偶校验位 = 1，使总数变为 4（偶数）。

常见错误

  • ❌ 比较时从最低位开始。大小比较必须从最高位（MSB）开始——正如你比较 352 和 349，先看百位。
  • ❌ 混淆奇校验和偶校验。记住：偶校验使 1 的总数为偶数。
  • ✅ 用 XOR 树实现奇偶校验：XOR 天然计算"1 的个数的奇偶性"。
