逻辑门的物理实现

01NOT / NAND / NOR 的 MOSFET 实现

🎯 为什么研究这个问题

在 Stage 1 中，我们学习了 MOSFET 晶体管——这种神奇的电控开关可以用电压信号控制电流的通断。但一个开关本身并不能"思考"。如果我们想让电路做出逻辑判断——比如"只有当 A 且 B 同时为真时，输出才为真"——就需要将多个晶体管组合起来，形成逻辑门（Logic Gate）。

逻辑门是数字电路的原子操作。就像化学中的原子组成分子，逻辑门组成加法器、存储器、乃至整个 CPU。理解逻辑门如何由晶体管物理实现，是从模拟电子学迈向数字计算的关键一步。这一节，我们从最简单的 NOT 门开始，逐步推导 NAND 和 NOR 门的 CMOS 实现。

🔍 抽象化方法

MOSFET 的核心行为可以抽象为两条规则：

NMOS（N 沟道 MOSFET）：栅极（Gate）为高电平时导通，充当连接到 GND 的开关。即"高电平打开，拉到地"。

PMOS（P 沟道 MOSFET）：栅极为低电平时导通，充当连接到 VDD 的开关。即"低电平打开，拉到电源"。

互补 CMOS（Complementary MOS）的设计哲学是：用 PMOS 构建上拉网络（Pull-Up Network, PUN）连接 VDD 与输出，用 NMOS 构建下拉网络（Pull-Down Network, PDN）连接输出与 GND。两个网络互补——当上拉导通时下拉截止，反之亦然——确保输出始终有确定的逻辑电平。

我们用真值表（Truth Table）来定义逻辑门的行为规范：

NOT 门：输入 0 → 输出 1；输入 1 → 输出 0。

NAND 门：只有当所有输入都为 1 时，输出为 0；否则输出为 1。

NOR 门：只要有任何一个输入为 1，输出为 0；所有输入都为 0 时输出为 1。

📐 公式推导 / 模型建立

NOT 门（反相器, Inverter）的构建：

NOT 门是最简单的 CMOS 逻辑门，仅需 1 个 PMOS + 1 个 NMOS。PMOS 源极接 VDD，漏极接输出；NMOS 源极接 GND，漏极接输出。两个晶体管的栅极共同接输入 \(A\)。

当 \(A = 0\)（低电平）：PMOS 栅极为低 → PMOS 导通，上拉网络将输出连接到 VDD → 输出 = 1。NMOS 栅极为低 → NMOS 截止，下拉网络断开。

当 \(A = 1\)（高电平）：PMOS 栅极为高 → PMOS 截止，上拉网络断开。NMOS 栅极为高 → NMOS 导通，下拉网络将输出连接到 GND → 输出 = 0。

\[Y = \overline{A}\]

NAND 门（与非门）的构建：

2 输入 NAND 门需要 2 个 PMOS（并联，构成上拉网络）+ 2 个 NMOS（串联，构成下拉网络），共 4 个晶体管。

下拉网络中两个 NMOS 串联：只有当 \(A=1\) 且 \(B=1\) 时，两个 NMOS 都导通，形成从输出到 GND 的通路 → 输出 = 0。

上拉网络中两个 PMOS 并联：只要 \(A=0\) 或 \(B=0\)，至少一个 PMOS 导通，输出连接到 VDD → 输出 = 1。

逐行验证真值表：

\(A=0, B=0\)：两个 PMOS 都导通 → 输出 = 1。两个 NMOS 都截止 → 无下拉通路。✓

\(A=0, B=1\)：PMOS-A 导通 → 输出 = 1。NMOS-B 导通但 NMOS-A 截止，串联链断开。✓

\(A=1, B=0\)：PMOS-B 导通 → 输出 = 1。NMOS-A 导通但 NMOS-B 截止。✓

\(A=1, B=1\)：两个 PMOS 都截止。两个 NMOS 都导通，串联链完整 → 输出 = 0。✓

\[Y = \overline{A \cdot B}\]

NOR 门（或非门）的构建：

NOR 门是 NAND 门的"对偶"：2 个 PMOS 串联（上拉）+ 2 个 NMOS 并联（下拉）。

下拉网络中两个 NMOS 并联：只要 \(A=1\) 或 \(B=1\)，至少一个 NMOS 导通 → 输出 = 0。

上拉网络中两个 PMOS 串联：只有当 \(A=0\) 且 \(B=0\) 时，两个 PMOS 都导通 → 输出 = 1。

\[Y = \overline{A + B}\]
⚠️ NMOS 拉低，PMOS 拉高——不要搞反！NMOS 导通时将输出拉向 GND（逻辑 0），PMOS 导通时将输出拉向 VDD（逻辑 1）。串联 NMOS 意味着"所有输入都为 1 才能拉低"（AND 条件），并联 NMOS 意味着"任意一个输入为 1 就拉低"（OR 条件）。PMOS 网络恰好相反。
🌐 实际应用

在现代芯片中的角色：当今的处理器芯片（如 Apple M 系列、AMD Zen 架构）集成了数百亿个 CMOS 晶体管，这些晶体管组成的逻辑门是所有计算的物理基础。

NOT 门（反相器）是最基本的逻辑单元，在时钟信号生成、信号缓冲、电平转换中无处不在。在 SRAM 存储单元中，两个交叉耦合的反相器构成一个锁存器（Latch），保存 1 bit 数据。

NAND 门与 NAND Flash：闪存（Flash Memory）之所以叫"NAND Flash"，正是因为其存储单元阵列的结构类似串联的 NAND 门。在 NAND Flash 中，多个浮栅晶体管串联连接，通过检测串联链的导通状态来读取数据。这种串联结构极大提高了存储密度。

NOR Flash的存储单元并联连接，读取速度快但密度低，常用于存储需要快速随机访问的固件代码（如 BIOS/UEFI）。

✏️ 例题分析与易错点

例题：给定一个 CMOS NAND 门，输入 \(A=1, B=0\)，分析每个晶体管的状态并确定输出。

✅ 正确分析：

上拉网络（PMOS 并联）：PMOS-A 栅极 = 1 → PMOS-A 截止。PMOS-B 栅极 = 0 → PMOS-B 导通，输出通过 PMOS-B 连接到 VDD。

下拉网络（NMOS 串联）：NMOS-A 栅极 = 1 → NMOS-A 导通。NMOS-B 栅极 = 0 → NMOS-B 截止。串联链不完整，输出无法拉低。

结论：输出 = VDD = 逻辑 1。与 NAND 真值表一致：\(\overline{1 \cdot 0} = \overline{0} = 1\)。

❌ 常见错误：混淆 NMOS 的串联与并联。有些初学者认为"只要一个 NMOS 导通，输出就被拉低"——这对并联 NMOS（NOR 门）成立，但对串联 NMOS（NAND 门）不成立。串联要求所有 NMOS 都导通才能形成完整的通路。

⚠️ 切勿让上拉和下拉同时导通！如果设计错误导致 PMOS 和 NMOS 同时导通，VDD 和 GND 之间会形成短路，产生巨大的"穿通电流"（Shoot-through Current），导致功耗激增甚至烧毁芯片。正确的 CMOS 设计确保互补——永远只有一个网络导通。

02NAND 是万能门（Universal Gate）

🎯 为什么研究这个问题

假设你是一个芯片工厂的工程师，需要在硅片上制造各种逻辑门：NOT、AND、OR、XOR……每种门的晶体管排列方式不同，制造流程也不同。如果能找到一种"万能门"——仅用这一种门就能构建任意逻辑功能——那么工厂只需要优化一种门的制造工艺，大幅简化生产流程、降低成本、提高良率。

这就是万能门（Universal Gate）的概念：一种门类型，能够单独实现所有布尔逻辑运算。NAND 门和 NOR 门都是万能门，而 AND、OR、NOT 单独使用时都不是万能门。这个看似理论化的性质，实际上深刻影响了整个半导体工业的设计方法学。

🔍 抽象化方法

万能门的定义：如果仅使用某一种逻辑门及其组合，可以实现任意布尔函数（Boolean Function），则称该门为万能门（Universal Gate）。

任意布尔函数都可以用 AND、OR、NOT 三种基本运算的组合来表达（这是布尔代数的完备性定理）。因此，要证明某个门是万能门，只需证明它能实现 NOT、AND、OR 这三种基本运算即可。

为什么 AND 不是万能门？AND 门的输出永远 ≤ 输入（从 1→1 或 1→0，不可能从 0→1 "创造"高电平），因此无法实现 NOT 的"反转"功能。同理，OR 门也无法实现反转。只有具有反相能力（Inverting Capability）的门才可能是万能门。

NAND（与非）和 NOR（或非）都自带反相功能（输出是输入的"非"某种组合），因此它们都是万能门的候选者。

📐 公式推导 / 模型建立

第一步：用 NAND 实现 NOT

将 NAND 门的两个输入连接在一起（短接）：

\[\text{NAND}(A, A) = \overline{A \cdot A} = \overline{A} = \text{NOT}(A)\]

只需 1 个 NAND 门即可实现 NOT。

第二步：用 NAND 实现 AND

先做 NAND，再对结果取 NOT（用上一步的方法）：

\[\text{AND}(A, B) = \overline{\overline{A \cdot B}} = \text{NOT}(\text{NAND}(A, B))\]

需要 2 个 NAND 门：第一个计算 \(\overline{A \cdot B}\)，第二个将结果取反。

第三步：用 NAND 实现 OR

利用德摩根定律（De Morgan's Law）：

\[A + B = \overline{\overline{A + B}} = \overline{\overline{A} \cdot \overline{B}}\]

分解为 NAND 操作：先分别取反 \(\overline{A} = \text{NAND}(A,A)\)，\(\overline{B} = \text{NAND}(B,B)\)，然后对两个结果做 NAND：

\[\text{OR}(A,B) = \text{NAND}\big(\text{NAND}(A,A),\;\text{NAND}(B,B)\big)\]

需要 3 个 NAND 门。

门数统计：

NOT = 1 个 NAND，AND = 2 个 NAND，OR = 3 个 NAND。由此证明 NAND 可以实现 NOT、AND、OR，因此 NAND 是万能门。同样的推导可以对 NOR 做一遍，证明 NOR 也是万能门。

⚠️ AND 和 OR 单独都不是万能门！它们缺乏反相能力。AND 门无论怎么组合，输出只要某个输入为 0 就为 0——永远无法从全 0 输入得到 1 输出（这是 NOT(0)=1 的要求）。必须要有至少一个能"反转"信号的门。
🌐 实际应用

标准单元库（Standard Cell Library）：在现代 VLSI 设计中，芯片制造商（如 TSMC、Samsung）提供的标准单元库以 NAND 和 NOR 门为核心。设计工具（EDA 工具如 Synopsys Design Compiler）在逻辑综合（Logic Synthesis）阶段，会将 RTL 代码自动映射到这些基本门单元上。

为什么偏好 NAND 而非 NOR？在 CMOS 工艺中，NMOS 的迁移率（Mobility）约为 PMOS 的 2-3 倍，意味着相同尺寸下 NMOS 的驱动能力更强。NAND 门的关键路径经过串联 NMOS（速度较快）和并联 PMOS，而 NOR 门的关键路径经过串联 PMOS（速度较慢）。因此，在同等面积约束下，NAND 门通常比 NOR 门速度更快，功耗更低。这就是为什么实际芯片设计中 NAND 门的使用频率远高于 NOR 门。

FPGA 与查找表（LUT）：FPGA（现场可编程门阵列）将万能门的思想推向了极致——其基本逻辑单元是查找表（Look-Up Table），本质上是一个小型存储器，可以实现任意 N 输入布尔函数。

✏️ 例题分析与易错点

例题：仅使用 NAND 门实现 XOR（异或）功能：\(Y = A \oplus B = A\overline{B} + \overline{A}B\)

✅ 正确解法：

XOR 的表达式变换：

\[A \oplus B = \overline{\overline{A \oplus B}} = \overline{\overline{A\bar{B} + \bar{A}B}}\]

利用 NAND 的性质，一种高效实现方式是：

令 \(P = \text{NAND}(A, B)\)，则 \(P = \overline{AB}\)。

令 \(Q = \text{NAND}(A, P) = \overline{A \cdot \overline{AB}}\)，令 \(R = \text{NAND}(B, P) = \overline{B \cdot \overline{AB}}\)，最后 \(Y = \text{NAND}(Q, R)\)。

展开为标准 NAND 链表达式：

\[A \oplus B = \text{NAND}\big(\text{NAND}(A, \text{NAND}(A,B)),\;\text{NAND}(B, \text{NAND}(A,B))\big)\]

这需要 4 个 NAND 门，是 XOR 的最优 NAND 实现。验证：当 \(A=1, B=0\) 时，\(\text{NAND}(1,0)=1\)，\(\text{NAND}(1,1)=0\)，\(\text{NAND}(0,1)=1\)，\(\text{NAND}(0,1)=1\)。最终 \(\text{NAND}(0,1)=1\)。✓

❌ 常见错误：不少学生直接用 \(\text{NAND}(A,B)\) 来代替 XOR——两者的真值表完全不同！NAND(0,0)=1 而 XOR(0,0)=0。记住：NAND 不等于 XOR。

03传播延迟（Propagation Delay）与功耗（Power）

🎯 为什么研究这个问题

在理想的逻辑世界中，门的输出在输入改变的瞬间就立刻跟着变化。但在真实的物理世界中，每个逻辑门都有一个微小但不可忽略的传播延迟（Propagation Delay）。当一个现代 CPU 包含数十亿个门，这些微小的延迟累积起来，就决定了芯片能跑多快——即最大时钟频率。

与此同时，每次逻辑门切换状态都会消耗能量。当数十亿个门以 GHz 频率不停翻转时，功耗（Power Dissipation）成为巨大的工程挑战。一颗高性能 CPU 可以消耗 100-300 瓦的功率——相当于一个小型电暖器！理解延迟和功耗的物理机制，是优化芯片性能的基础。

🔍 抽象化方法

传播延迟的定义：

\(t_{pHL}\)：输出从高电平跳变到低电平的延迟（High-to-Low），测量输入变化到输出跨过 50% VDD 的时间。

\(t_{pLH}\)：输出从低电平跳变到高电平的延迟（Low-to-High）。

平均传播延迟：

\[t_p = \frac{t_{pHL} + t_{pLH}}{2}\]

功耗的两大分量：

静态功耗（Static Power） \(P_{static}\)：即使电路不切换，晶体管的漏电流也会消耗能量。在理想 CMOS 中可忽略，但在纳米工艺中越来越显著。

动态功耗（Dynamic Power） \(P_{dynamic}\)：每次门的输出切换时，需要对负载电容充放电，消耗能量。这是数字电路功耗的主要来源。

📐 公式推导 / 模型建立

传播延迟的 RC 模型：

将导通的 MOSFET 近似为一个等效电阻 \(R_{on}\)，将门的输出节点上的所有电容（包括自身漏极电容、连线电容、下一级栅极电容）合并为负载电容 \(C_L\)。输出电压的变化就是一个 RC 电路的充放电过程：

\[V_{out}(t) = V_{final} + (V_{initial} - V_{final}) \cdot e^{-t/\tau}, \quad \tau = R_{on} \cdot C_L\]

传播延迟定义为输出达到 50% 点的时间，即 \(e^{-t_p/\tau} = 0.5\)：

\[t_p = \tau \cdot \ln 2 \approx 0.69 \cdot R_{on} \cdot C_L\]

这就是著名的 0.69RC 延迟公式——数字电路设计中最基本的延迟估算工具。

动态功耗的推导：

每次输出从 0 跳到 1（充电）：电源对 \(C_L\) 充电，输入能量 \(E_{charge} = C_L \cdot V_{DD}^2\)。其中一半存储在电容中 \(\frac{1}{2}C_L V_{DD}^2\)，另一半消耗在 PMOS 的等效电阻上。

每次输出从 1 跳到 0（放电）：电容通过 NMOS 放电到 GND，存储的 \(\frac{1}{2}C_L V_{DD}^2\) 全部消耗在 NMOS 上。

因此，一个完整的 0→1→0 周期总共消耗能量 \(C_L \cdot V_{DD}^2\)。如果以频率 \(f\) 切换，且活动因子（Activity Factor）为 \(\alpha\)（表示每个周期实际翻转的概率），则：

\[P_{dynamic} = \alpha \cdot C_L \cdot V_{DD}^2 \cdot f\]
⚠️ 动态功耗与 \(V_{DD}^2\) 成正比！这意味着将电压降低一半，动态功耗减少到原来的 \(1/4\)。这就是为什么现代芯片的工作电压从早期的 5V 逐步降低到 1V 甚至更低。但降压同时会增加延迟（因为 \(R_{on}\) 增大），这就是速度与功耗之间的根本权衡（Speed-Power Tradeoff）。
🌐 实际应用

关键路径（Critical Path）决定时钟频率：CPU 中从一个寄存器到下一个寄存器之间最长的门延迟累积路径叫做"关键路径"。时钟周期必须大于关键路径延迟，否则数据来不及稳定。优化关键路径是芯片后端设计的核心工作。

电压频率动态调整（DVFS）：现代处理器可以动态调整 \(V_{DD}\) 和 \(f\)：轻负载时降低电压和频率（省电），重负载时提高电压和频率（提速）。笔记本电脑从满载到空闲，CPU 频率可能从 5 GHz 降到 0.8 GHz，功耗从 65W 降到不到 5W。

时钟门控（Clock Gating）：对于不活跃的模块，关闭其时钟信号，使 \(\alpha = 0\)，彻底消除动态功耗。这是现代 SoC 功耗管理的关键技术。

散热设计功耗（TDP）：芯片厂商标注的 TDP（如 Intel Core i9 的 125W TDP）是散热系统需要处理的最大持续功耗，主要由动态功耗贡献。

✏️ 例题分析与易错点

例题 1：计算传播延迟

某 CMOS 门的 \(R_{on} = 1\,\text{k}\Omega\)，负载电容 \(C_L = 50\,\text{fF}\)，求传播延迟 \(t_p\)。

✅ 解答：

\[t_p = 0.69 \times R_{on} \times C_L = 0.69 \times 1 \times 10^3 \times 50 \times 10^{-15}\] \[= 0.69 \times 50 \times 10^{-12} = 34.5\,\text{ps}\]

例题 2：计算动态功耗

某芯片有 \(10^9\) 个门，每个门的负载电容 \(C_L = 10\,\text{fF}\)，\(V_{DD} = 1\,\text{V}\)，时钟频率 \(f = 3\,\text{GHz}\)，平均活动因子 \(\alpha = 0.1\)，求总动态功耗。

✅ 解答：

\[P_{total} = N \cdot \alpha \cdot C_L \cdot V_{DD}^2 \cdot f\] \[= 10^9 \times 0.1 \times 10 \times 10^{-15} \times 1^2 \times 3 \times 10^9\] \[= 10^9 \times 0.1 \times 10^{-14} \times 3 \times 10^9 = 3\,\text{W}\]

（这只是动态功耗部分，实际芯片还有静态漏电功耗。）

❌ 常见错误：忘记 \(V_{DD}\) 是平方关系。有学生计算降压后的功耗时直接用线性比例——将电压减半后说功耗变为 1/2，实际应为 1/4。

⚡ 快速检测
如果将 \(V_{DD}\) 从 1.0 V 降低到 0.7 V，动态功耗变为原来的多少倍？

04互补 CMOS（CMOS）逻辑家族

🎯 为什么研究这个问题

在数字电路的发展历史中，出现过多种逻辑家族（Logic Family）：RTL（电阻-晶体管逻辑）、DTL（二极管-晶体管逻辑）、TTL（晶体管-晶体管逻辑）、ECL（发射极耦合逻辑）、CMOS（互补 MOS 逻辑）。为什么最终是 CMOS 几乎一统天下？

答案在于 CMOS 独特的互补结构带来了一个革命性优势：几乎为零的静态功耗。当芯片集成度从几百个晶体管暴增到数百亿个，这个优势变得不可替代。理解 CMOS 为什么赢，就是理解现代半导体产业的底层逻辑。

🔍 抽象化方法

CMOS 的四大核心特性：

1. 互补结构（Complementary Structure）：上拉网络由 PMOS 构成，下拉网络由 NMOS 构成。在任何稳态下，恰好只有一个网络导通，另一个截止。

2. 无直流通路（No DC Path）：由于互补导通，稳态下 VDD 到 GND 之间没有直流电流通路，静态功耗仅为晶体管漏电流（理想情况下为零）。

3. 轨到轨输出摆幅（Rail-to-Rail Swing）：输出高电平 \(V_{OH} = V_{DD}\)，输出低电平 \(V_{OL} = 0\,\text{V (GND)}\)。输出可以完全摆到电源轨，不像 TTL 那样 \(V_{OH} \approx 3.4\,\text{V}\)（低于 VDD=5V）。

4. 高噪声容限（High Noise Margin）：正因为轨到轨摆幅，CMOS 的噪声容限（Noise Margin）很大，抗干扰能力强。

📐 公式推导 / 模型建立

电压传输特性（VTC, Voltage Transfer Characteristic）：

VTC 是 CMOS 反相器最重要的静态特性曲线，描述输出电压 \(V_{out}\) 与输入电压 \(V_{in}\) 的关系。

对 CMOS 反相器，VTC 呈现为一条陡峭的 S 形（或反 S 形）曲线，有五个关键区域：

区域 1：\(V_{in} < V_{IL}\) — PMOS 完全导通，NMOS 截止，\(V_{out} = V_{OH} \approx V_{DD}\)。

区域 2：\(V_{in} \approx V_{IL}\) — NMOS 开始导通（进入饱和区），PMOS 仍在线性区。

区域 3：\(V_{in} \approx V_{DD}/2\)（切换点）— 两个晶体管都在饱和区，\(V_{out}\) 急剧下降，增益最大。

区域 4：\(V_{in} \approx V_{IH}\) — PMOS 开始截止，NMOS 在线性区。

区域 5：\(V_{in} > V_{IH}\) — NMOS 完全导通，PMOS 截止，\(V_{out} = V_{OL} \approx 0\)。

噪声容限（Noise Margin）的计算：

\[NM_H = V_{OH} - V_{IH}\] \[NM_L = V_{IL} - V_{OL}\]

其中 \(V_{IL}\) 和 \(V_{IH}\) 是 VTC 曲线上斜率（增益）为 −1 的两个点。

与 TTL 的静态功耗对比：

TTL 门内部有从 VDD 经电阻到 GND 的常导通电流路径，每个门的静态电流约 1-4 mA。如果芯片集成 \(10^9\) 个 TTL 门：

\[P_{static,\text{TTL}} = 10^9 \times 2\,\text{mA} \times 5\,\text{V} = 10^{10}\,\text{W} = 10\,\text{GW}\]

这显然是不可能的——所以 TTL 技术根本无法支持大规模集成。而 CMOS 的静态电流在微安甚至纳安级别，使大规模集成成为可能。

⚠️ "CMOS 零静态功耗"是一种理想化！在 45nm 以下的先进工艺中，晶体管的亚阈值漏电流（Subthreshold Leakage）和栅极隧穿电流（Gate Tunneling）变得不可忽略。例如在 5nm 工艺中，漏电功耗可能占总功耗的 30-50%。这就是为什么芯片设计中需要功率门控（Power Gating）等技术来关断不活跃模块的供电。
🌐 实际应用

CMOS 工艺节点的演进：半导体行业按照摩尔定律（Moore's Law）持续缩小晶体管尺寸，从早期的 10μm 工艺到如今的 3nm/2nm 工艺。每一代工艺节点都提升了集成度和性能，同时降低了每个晶体管的功耗。

74HC 系列：74HC（High-speed CMOS）系列是经典的 CMOS 逻辑芯片家族，与 TTL 的 74LS 系列引脚兼容，但功耗低得多。至今仍广泛用于教学和小规模数字电路设计。

电池供电设备：CMOS 的低功耗特性使其成为电池供电设备的唯一选择。从手表、手机到物联网传感器，所有需要长时间电池续航的设备都依赖 CMOS 技术。一颗 CMOS 实时时钟芯片的待机电流仅有几十纳安，一枚纽扣电池可以驱动它运行数年。

工艺节点里程碑：180nm（2000 年，互联网时代初期）→ 65nm（2006 年，智能手机起步）→ 14nm（2014 年，FinFET 开始）→ 7nm（2018 年，EUV 光刻）→ 3nm（2022 年，GAA 晶体管）。每一代都是 CMOS 技术的延续和革新。

✏️ 例题分析与易错点

例题：某 CMOS 反相器的参数为 \(V_{DD} = 3.3\,\text{V}\)，\(V_{IL} = 1.4\,\text{V}\)，\(V_{IH} = 1.8\,\text{V}\)，\(V_{OL} = 0.1\,\text{V}\)，\(V_{OH} = 3.2\,\text{V}\)。计算高电平和低电平噪声容限。

✅ 解答：

\[NM_H = V_{OH} - V_{IH} = 3.2 - 1.8 = 1.4\,\text{V}\] \[NM_L = V_{IL} - V_{OL} = 1.4 - 0.1 = 1.3\,\text{V}\]

这意味着高电平信号可以承受高达 1.4V 的噪声干扰而不被误判，低电平信号可以承受 1.3V 的噪声干扰。噪声容限大约是 \(V_{DD}\) 的 40%，这是非常健壮的抗干扰能力。

对比 TTL 参数：\(V_{DD}=5\,\text{V}\)，\(V_{OH}=2.7\,\text{V}\)，\(V_{OL}=0.4\,\text{V}\)，\(V_{IH}=2.0\,\text{V}\)，\(V_{IL}=0.8\,\text{V}\)。TTL 的噪声容限：\(NM_H = 2.7-2.0 = 0.7\,\text{V}\)，\(NM_L = 0.8-0.4 = 0.4\,\text{V}\)。CMOS 的噪声容限远优于 TTL。

❌ 常见错误：混淆 \(NM_H = V_{OH} - V_{IH}\) 和 \(NM_L = V_{IL} - V_{OL}\) 的公式。一些学生把 \(NM_H\) 写成 \(V_{IH} - V_{OH}\)（结果为负数），或者把 \(V_{OL}\) 和 \(V_{IL}\) 搞混。记忆技巧：噪声容限 = "输出保证" − "输入要求"，结果必须为正。
